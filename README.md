Each folder represents a core component of the Spikey ecosystem. Please use the `en.json` file in each directory as your master reference:

* **`/spikeygames`**: Terms for the **Memeverse**, the **Souls system** (SBTs/EXP), and fair games powered by **Supra‚Äôs dVRF**.

## üõ† How to Contribute (The Fork & PR Workflow)

To contribute, you don't need direct access to the main code. Instead, you propose changes that our team will review and approve:

1.  **Fork the Repo**: Click the "Fork" button at the top right of this page to create your own copy of the repository.
2.  **Create your Translation**:
    * Go to the folder you want to translate.
    * Copy the contents of `en.json`.
    * Create a new file named with your language code (e.g., `ha.json` for Hausa).
    * Paste the content and translate only the **values** (the text on the right side). **Do not change the keys** (the text on the left).
3.  **Commit and Push**: Save your changes in your forked repository.
4.  **Submit a Pull Request (PR)**:
    * Go to the "Pull Requests" tab in the original Spikey repository.
    * Click "New Pull Request" and select your fork.
    * Add a description of which language you added/updated.

---

## ‚úÖ The Approval Process

Once you submit a Pull Request, our technical team will review it:
* **JSON Validation**: We check that the file format is correct (no missing commas or quotes).
* **Context Check**: We ensure technical terms (like "Bonding Curve" or "Fungible Assets") are translated accurately within the Spikey context.
* **Approval**: Once verified, we will **Approve and Merge** your contribution into the main branch, making it live for the ecosystem!

## üí° Important Context for Translators

To provide the best translations, keep these core principles in mind:
* **Multisig is a Vault**: Ensure users understand that the Multisig is for storage and cannot interact directly with other dApps.
* **Souls are EXP**: In the Memeverse, "Souls" are non-transferable tokens that act as experience points for leveling up Digital Assets.
* **Bonding Curve**: This is the mathematical formula used in the Launchpad to determine token price based on supply and demand.
* **Zap Feature**: In Spikey Swap, "Zap" allows users to provide liquidity using only one type of token instead of two.
